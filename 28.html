Core Data Content of the Frontend Module

Instant Action: Focus on comprehension and complexity reduction
1
Icon - Logo (Vector: SVG)
Principle: Instant Comprehension via Icon
High contrast, clear link between icon and action. Peripheral vision. Clicking leads exactly to the intended internal action.
2
Compressed Imperative Description (Instruction)
Principle: Reduction of Cognitive Load
Examples: "Report fault", "Issue invoice", "Missing in stock". Identification and removal of inefficient symbols.
3
Attention Driver (Productivity Weight)
Principle: Static Numbering (1-7)
The most productive task is number 1. Set by leadership (Shift Leaders, COO, CEO) based on corporate importance.
4
Dynamic Status Indicator (Live Status)
Principle: Elimination of Unnecessary Clicks
Notifications (Motion, sound, color) directly on the button. The user clicks only to resolve the status, not to check it.
Data Model Summary
Icon (Eye sees) → Description (Brain understands) → Weight (Importance) → Status (System calls for help)
